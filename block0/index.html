Для обучения и выполнения заданий на PHP потребуется PHP интерпретатор и настроенный локальный web-сервер. Для Linux можно воспользоваться следующей инструкцией (в целом применимо и к более новым версиям ОС): Инструкция по установке LEMP Альтернатива для windows: OpenServer. Для Vue.js и Nuxt.js проектов: для создания web-приложений на JS-фреймворках потребуется установленный в окружении node.js.

Все задания лучше выкладывать в репозиторий на github и дать ссылку наставнику. Допускается все задания держать в одном репозитории в разных каталогах и файлах. Задание для итогового приложения

    Требуется создать веб-проект - сервис микроблогов. Назовём его Bitter. Реализовать его с Nuxt.js (либо Vue.js) на фронтенде и Laravel в качестве бэкенда с API.

    Есть пользователи, у них есть возможность создать короткий пост, до 280 символов, и “отметить” в тексте других пользователей или хэштег. “Отметка” ставится путем добавления в текст специального слова - @nickname, где “nickname” - это логин пользователя в системе. А хэштег ставится добавлением слова с решёткой в начале - #хэштег. Например, пост “Это мои друзья: @gena, @vlad, @sveta #друзья” будет означать, что отмечены пользователи с логином “gena”, “vlad” и “sveta”, и добавлен хэштег #друзья.

    Пользователи подписываются на обновления других пользователей.

    Необходимо реализовать три страницы. По сути это почти одинаковые списки постов, только они формируются по разным признакам.

    Первая страница - главная, своя лента. Сюда должны попадать посты пользователей, на которых мы подписаны, плюс посты, на которых мы отмечены, плюс наши посты. В списке над каждым постом вывести автора и время публикации. Затем вывести сам текст поста. Порядок - от самой новой к самой старой. Можно использовать пагинатор, а можно просто вывести штук 30 последних постов.

    Вторая страница - список постов пользователя. (Например, /valera) Нужно вывести посты, добавленные пользователем, а также посты, на которых он отмечен.

    Третья страница - список постов по хэштегу. (Например, /hash/friends) Нужно вывести посты, в тексте которых есть данный хэштег.

    И добавление поста. Нужно использовать общий layout для всех страниц, вверху которого будет форма добавления поста, по сути просто текстовое поле. Авторизацию делать не нужно сейчас, будем считать, что текущий авторизованный пользователь всегда один и тот же, например User с id 1. Если останется время, можно будет реализовать авторизацию.

    Если останется время, можно развить проект - добавить кнопки подписки на пользователя, открытые/закрытые аккаунты, личные сообщения, сокеты для личных сообщений, уведомления о новых постах в ленте. На что хватит времени.

    Нужно помнить и учитывать, что в таком сервисе может быть несколько миллионов пользователей и строить работу с БД исходя из этого. То есть, для выбора постов с поиском по отметке или хэштегу, нельзя использовать полнотекстовый поиск по самому тексту поста. Нужно реализовать отдельную таблицу хэштегов. И нужно создать средствами laravel отношения постов и хэштегов (Many to Many). Для выборок постов можно будет использовать соответствующие настроенные связи.

    Будем отталкиваться от дизайна твиттера. Всё-таки функционал примерно заимствуется оттуда. Но будем допускать и приветствовать своё видение интерфейса.

    Углубляться в HTML и CSS сильно не нужно, делать всё просто на базовом уровне в соответствии со своими компетенциями в области фронтенда.
